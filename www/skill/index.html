CSP Agent Skill for Claude Code

Audit your web application and generate a Content-Security-Policy header — directly from your terminal. This Claude Code skill scans your codebase, identifies every inline script, style, and external resource, then recommends (or implements) a strict CSP tailored to your stack.

Get the Skill — Starting at $99

What It Does

Run /csp in Claude Code and the skill will:

  1. Detect your tech stack
  2. Audit every CSP-relevant resource
  3. Catalog inline code
  4. Recommend the optimal fix for each
  5. Generate a complete CSP header

Run /csp implement and it will apply the policy to your project — adding the header, wiring up nonce generation, and refactoring inline event handlers where needed.

Features

  1. Builtin Reference Files - the skill comes with dozens of reference files providing unique guidance that you won't get with a one shot attempt.
  2. Create Strong Policies - there are plenty of ways to build a weak CSP policy, the skill guides towards stronger more secure policy implementations.
  3. Eliminate 'unsafe-inline' - The single biggest CSP weakness is 'unsafe-inline' in script-src. This skill is specifically designed to help you remove it
  4. Sensible Defaults - The implement skill defaults to Content-Security-Policy-Report-Only so you can monitor violations before enforcing.
  5. SKILL.md - a SKILL.md file designed to work with claude code.

Usage

/csp or /csp audit Audit the project and generate a recommended CSP header. No files are modified.
/csp implement Audit the project, then apply the CSP to the appropriate config files, wire up nonce generation, and refactor inline event handlers where needed.

The skill defaults to Content-Security-Policy-Report-Only so you can monitor violations before enforcing.

What You Get

The audit report includes:

  • Detected stack — frameworks, servers, and hosting platforms identified in your project.
  • Resource origins table — every external origin discovered per directive (script-src, style-src, img-src, etc.) with file references.
  • Inline code inventory — deduplicated table of every inline script, style, and event handler with occurrence counts, server-variable detection, and the recommended remediation for each.
  • Recommended CSP header — the full policy in both multi-line and single-line format, ready to deploy.
  • Warnings — flags for 'unsafe-eval', wildcards, assumptions made, and browser support considerations.
  • Next steps — how to deploy with Report-Only, monitor violations, and switch to enforcing mode.
